# خلاصه تغییرات انجام شده برای رفع مشکل ذخیره‌سازی داده‌ها

## تاریخ: 1404/08/05 (26 اکتبر 2025)

## مشکل اصلی
داده‌های ویرایش شده در بخش "مدیریت استان‌ها" ذخیره نمی‌شدند و بعد از بارگذاری مجدد صفحه، تغییرات از بین می‌رفتند.

## علت مشکل
1. **API Routes ناقص بود**: در backend هیچ endpoint برای ذخیره دسته‌ای (batch update) استان‌ها و مراکز درمانی وجود نداشت
2. **URL های Hardcoded**: Frontend از URLهای hardcoded (`localhost:6000`) استفاده می‌کرد که با پورت واقعی backend (5001) مطابقت نداشت
3. **عدم استفاده از Vite Proxy**: تنظیمات proxy در Vite به درستی استفاده نمی‌شد

## تغییرات انجام شده

### 1️⃣ اضافه کردن API Routes جدید
**فایل**: `/backend/routes/territories.js`

دو endpoint جدید اضافه شد:

#### 🔹 `PUT /api/territories/provinces`
ذخیره دسته‌ای اطلاعات استان‌ها شامل:
- نام استان
- کارمند تخصیص یافته
- سهم بازار محصولات

**نمونه درخواست**:
```json
{
  "provinces": [
    {
      "id": "AZ",
      "name": "آذربایجان شرقی",
      "assignedTo": 1,
      "marketShare": {
        "6": 8.5,
        "7": 7.3,
        "8": 6.2
      }
    }
  ]
}
```

#### 🔹 `PUT /api/territories/medical-centers`
ذخیره دسته‌ای اطلاعات مراکز درمانی تهران شامل:
- نام مرکز درمانی
- کارمند تخصیص یافته
- سهم بازار محصولات

**نمونه درخواست**:
```json
{
  "centers": [
    {
      "id": "mc_emam_khomeini",
      "name": "بیمارستان امام خمینی",
      "marketShare": {
        "6": 12.5,
        "7": 9.8,
        "8": 11.2
      }
    }
  ]
}
```

### 2️⃣ اصلاح URL های Frontend
تمام فایل‌های زیر اصلاح شدند تا از URL های نسبی (relative URLs) استفاده کنند:

- ✅ `/contexts/AppContext.tsx` - تمام درخواست‌های API
- ✅ `/components/ManagementView.tsx` - ویرایش محصولات
- ✅ `/components/AutoTargetingView.tsx` - محاسبات هدف‌گذاری
- ✅ `/services/api.js` - سرویس API مرکزی

**قبل از تغییر**:
```javascript
fetch('http://localhost:6000/api/products')
```

**بعد از تغییر**:
```javascript
fetch('/api/products')  // استفاده از Vite proxy
```

### 3️⃣ تنظیمات Vite Proxy
**فایل**: `/vite.config.ts`

```typescript
proxy: {
  '/api': {
    target: 'http://localhost:5001',  // Backend port
    changeOrigin: true,
    secure: false
  }
}
```

## نتیجه نهایی

### ✅ مشکلات برطرف شده:
1. **ذخیره‌سازی استان‌ها**: تغییرات سهم بازار محصولات در استان‌ها با موفقیت ذخیره می‌شود
2. **ذخیره‌سازی مراکز درمانی**: تغییرات سهم بازار محصولات در مراکز درمانی تهران با موفقیت ذخیره می‌شود
3. **ویرایش محصولات**: نام و قیمت محصولات قابل ویرایش و ذخیره‌سازی است
4. **پایداری داده‌ها**: تمام تغییرات در دیتابیس SQLite ذخیره می‌شوند و بعد از بارگذاری مجدد صفحه حفظ می‌مانند

### 🗄️ جداول دیتابیس درگیر:
- `territories` - اطلاعات استان‌ها و مراکز درمانی
- `territory_market_shares` - سهم بازار محصولات در هر منطقه
- `products` - اطلاعات محصولات (نام، قیمت)

### 🔒 ویژگی‌های امنیتی:
- استفاده از Transaction در دیتابیس برای جلوگیری از داده‌های ناقص
- بررسی Foreign Key constraints
- Validation ورودی‌ها

## نحوه استفاده

### برای کاربران:
1. وارد بخش "مدیریت استان‌ها" شوید
2. در جدول استان‌ها، درصد سهم بازار هر محصول را وارد کنید
3. دکمه "ذخیره تغییرات سهم بازار" را بزنید
4. تغییرات بلافاصله در دیتابیس ذخیره می‌شوند
5. با بارگذاری مجدد صفحه، داده‌ها همچنان موجود هستند
